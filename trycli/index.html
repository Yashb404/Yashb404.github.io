TryCli Studio

"Orchestrating ephemeral, browser-based Docker sandboxes to eliminate configuration drift and accelerate developer onboarding for CLI tools."

The "npm install" Barrier

Developer adoption drops significantly during initial setup. Static documentation forces users to clone repos and resolve dependencies locally, leading to "works on my machine" errors. Existing solutions like Codespaces are too heavy for simple demos.

Market Gap

Current tools (Repl.it, StackBlitz) focus on full-stack web apps, lacking specialized support for raw terminal/CLI interactions and custom Linux binaries.

Our Innovation

Zero-config, ephemeral Docker snapshots spawned in < 500ms using a high-performance Rust orchestrator.

Tech Stack

Client (WASM)

Built with Leptos. Compiles to WebAssembly for near-native performance. Features Xterm.js for rendering the TTY.

Orchestrator

Axum (Rust) backend managing Docker via the Bollard crate. Handles WebSocket streams for stdin/stdout.

Performance Analysis

Environment Startup ~300ms
Memory Footprint 15MB

Compared to traditional VM provisioning (30s+) or Node.js backends, the Rust + Docker snapshot architecture delivers instant interactive terminals with minimal resource overhead.
